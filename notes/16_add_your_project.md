# 16 Add your project
- it is time to play with something more real world
- choose one of your projects and copy it into a folder for testing
- this folder will serve as a 'playground', so if something goes tits up you will be safe having the original files on different place
- after copying your files into the playground folder issue `git init`
- there will be a lot of file to add initially but just add them `git add .`
- and finally make your initial commit `git commit -m "initial commit"`
- you can have many separate repositories and each one will track its changes separately 
- as you navigate into your directories you will find there your .git folder and changes inside
- general advice is not to nest repositories, just keep things separate
