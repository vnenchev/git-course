# 55 Local and remote repositories

- local git don't require a network connection
- remote repositories (remotes) are hosted by servers over a network
- remotes allow easy collaboration
- push local repository changes to the remote
- fetch remote changes to the local repository
- git is designed to be distributed version control
- all repositories have equal authority
- one remote is often designated as the primary repository for collaboration
- you push your commits, your collaborators fetch the changes, and also in opposite direction
