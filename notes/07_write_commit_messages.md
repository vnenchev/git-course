# 7 Write commit messages
- a short single line summary somewhere below < 50 characters
- optionally followed by blank line and a more complete description, so your summary is something like a short title
- keep each line less than 72 characters, write in present tense
- it is good to refer to specific place if bug tracking with a number or kind of ID, ask for naming nomenclature
- is it helpful to designate your commit if you have a name system for doing this like:
    - `[css,js]` to point out files
    - `bugfix:` to show bugfix
    - `#384` - like issue number
